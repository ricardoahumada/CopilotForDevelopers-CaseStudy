# Prompt: NO Efectivo para SDD (Vibe Coding Tradicional)

**Contexto de uso:** Ejemplo de prompt que NO se debe usar en Spec-Driven Development por falta de especificidad.

**Prompt no específico:**

```
"Crea una función para validar emails"
```

**Problemas identificados:**

| Problema | Descripción |
|----------|-------------|
| Sin contexto de proyecto | No indica stack tecnológico |
| Sin especificaciones | No define reglas de validación |
| Sin ubicación | No dice dónde colocar el código |
| Sin criterios | No hay forma de validar el resultado |
| Sin convenciones | No indica estilo de código |

**Lección aprendida:** En SDD, los prompts deben incluir especificaciones completas, contexto del proyecto y criterios de aceptación claros.
