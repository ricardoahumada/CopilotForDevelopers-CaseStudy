# Checklist: Migración a SDD en Proyecto Brownfield

**Contexto de uso:** Checklist para implementar Spec-Driven Development en un proyecto existente.

**Checklist de implementación SDD:**

```markdown
## Checklist: Migración a SDD

### Fase 1: Preparación
- [ ] Evaluar complejidad del proyecto
- [ ] Identificar áreas prioritarias para SDD
- [ ] Obtener buy-in del equipo

### Fase 2: Configuración Base
- [ ] Instalar Spec Kit
- [ ] Crear AGENTS.md inicial
- [ ] Configurar estructura specs/
- [ ] Crear constitution.md con principios

### Fase 3: Documentación
- [ ] Documentar arquitectura actual
- [ ] Identificar patrones de código existentes
- [ ] Crear templates de refactoring
- [ ] Documentar convenciones en AGENTS.md

### Fase 4: Pilot
- [ ] Elegir feature pequeño para pilot
- [ ] Aplicar workflow SDD completo
- [ ] Medir resultados y ajustar

### Fase 5: Rollout
- [ ] Entrenar al equipo
- [ ] Aplicar a más features
- [ ] Iterar en proceso
```

**Fases de migración:**

| Fase | Duración estimada | Objetivo |
|------|-------------------|----------|
| Preparación | 1 semana | Evaluación y buy-in |
| Configuración Base | 2-3 días | Setup técnico |
| Documentación | 1 semana | Capturar conocimiento |
| Pilot | 1-2 sprints | Validar proceso |
| Rollout | Continuo | Adopción gradual |

**Nota:** Para proyectos greenfield (nuevos) como PortalEmpleo, comenzar directamente en Fase 2.
