# Documentación de Configuración de GitHub Copilot y GitHub Copilot Chat en VS Code

## Introducción a la Configuración en VS Code

Visual Studio Code permite configurar los plugins de GitHub Copilot y GitHub Copilot Chat mediante el archivo `settings.json`, que puede editarse a través de la Paleta de Comandos con el comando **Preferences: Open User Settings (JSON)** para configuraciones de usuario, o **Preferences: Open Workspace Settings (JSON)** para configuraciones específicas del proyecto. Esta documentación proporciona una referencia completa de todos los parámetros configurables disponibles, organizados por categoría funcional, con sus tipos de datos, valores por defecto y descripciones detalladas que te permitirán optimizar el comportamiento de Copilot según las necesidades específicas de tu flujo de trabajo.

La configuración en VS Code opera en tres niveles jerárquicos: las configuraciones de usuario que se aplican a todas las instancias de VS Code, las configuraciones de workspace que aplican a todo el proyecto actual, y las configuraciones de folder que aplican a carpetas específicas dentro del proyecto. Los parámetros de GitHub Copilot siguen esta misma jerarquía, donde las configuraciones de nivel inferior pueden sobrescribir las de nivel superior cuando sea necesario.

Es importante destacar que algunos parámetros requieren recargar la ventana de VS Code para que los cambios surtan efecto, mientras que otros se aplican de manera inmediata. Esta distinción se indicará en la descripción de cada parámetro cuando sea relevante para la configuración.

## Configuraciones Generales de GitHub Copilot

Las configuraciones generales controlan el comportamiento fundamental del plugin de Copilot, incluyendo la habilitación global, la activación por lenguaje de programación y las opciones de presentación en la interfaz de usuario.

El parámetro principal para controlar la activación de Copilot es `github.copilot.enable`, que acepta un objeto que permite habilitar o deshabilitar las sugerencias inline para lenguajes de programación específicos. Este parámetro es fundamental para proyectos donde deseas activar Copilot solo para ciertos tipos de archivos, excluyendo aquellos donde las sugerencias automáticas podrían ser contraproducentes o donde el equipo ha decidido no utilizar asistencia de IA.

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `github.copilot.enable` | objeto | `{ "*": true, "plaintext": false, "markdown": false, "scminput": false }` | Objeto que define qué lenguajes tienen habilitadas las sugerencias inline. El valor `"*"` actúa como comodín para todos los lenguajes no especificados individualmente. |
| `editor.inlineSuggest.enabled` | booleano | `true` | Controla si las sugerencias inline están habilitadas globalmente en el editor. Debe ser `true` para que Copilot muestre sugerencias de código. |
| `github.copilot.editor.enableCodeActions` | booleano | `true` | Determina si los comandos de Copilot aparecen como Acciones de Código en el menú contextual del editor. |
| `github.copilot.renameSuggestions.triggerAutomatically` | booleano | `true` | Genera sugerencias automáticas para renombrar símbolos cuando se utiliza la funcionalidad de rename de VS Code. |
| `chat.commandCenter.enabled` | booleano | `true` | Controla si se muestra el menú de Chat en la barra de título de VS Code, proporcionando acceso rápido a las funcionalidades de Copilot Chat. |
| `github.copilot.nextEditSuggestions.enabled` | booleano | `true` | Habilita las Next Edit Suggestions (NES), una funcionalidad que sugiere ediciones de código basadas en el contexto y patrones detectados. |

**Ejemplo de configuración por lenguaje:**

```json
{
    "editor.inlineSuggest.enabled": true,
    "github.copilot.enable": {
        "*": true,
        "yaml": false,
        "python": true,
        "javascript": true,
        "typescript": true,
        "csharp": true,
        "java": true
    }
}
```

Esta configuración habilita Copilot para todos los lenguajes excepto YAML, con la posibilidad de sobrescribir esta configuración a nivel de workspace o folder para proyectos específicos que requieran comportamientos diferentes.

## Configuraciones de Sugerencias Inline Avanzadas

Las configuraciones avanzadas de sugerencias inline permiten personalizar aspectos técnicos del comportamiento de las sugerencias, incluyendo el renderizado de sintaxis, tiempos de espera y comportamientos de edición. Estas opciones son especialmente útiles para desarrolladores que desean optimizar el rendimiento de VS Code o personalizar la experiencia de usuario según sus preferencias.

El parámetro `editor.inlineSuggest.syntaxHighlightingEnabled` controla si las sugerencias inline muestran resaltado de sintaxis, lo cual facilita la identificación del código sugerido pero puede tener un impacto menor en el rendimiento en proyectos muy grandes. Por otro lado, `editor.inlineSuggest.minShowDelay` permite configurar un tiempo de espera en milisegundos antes de mostrar las sugerencias, útil para evitar distracciones cuando el usuario está escribiendo rápidamente.

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `editor.inlineSuggest.syntaxHighlightingEnabled` | booleano | `true` | Habilita el resaltado de sintaxis para las sugerencias inline, haciendo más legible el código sugerido. |
| `editor.inlineSuggest.minShowDelay` | número | `0` | Tiempo en milisegundos que VS Code espera antes de mostrar una sugerencia inline. Valores mayores reducen las distracciones. |
| `editor.inlineSuggest.edits.allowCodeShifting` | cadena | `"always"` | Controla si NES puede desplazar código existente para mostrar sugerencias. Valores: `"always"`, `"never"`, `"auto"`. |
| `editor.inlineSuggest.edits.renderSideBySide` | cadena | `"auto"` | Configura si NES muestra sugerencias grandes una al lado de la otra o debajo. Valores: `"always"`, `"never"`, `"auto"`. |
| `github.copilot.nextEditSuggestions.fixes` | booleano | `true` | Habilita NES basado en diagnósticos del editor, como importaciones faltantes o errores de sintaxis. |

**Ejemplo de configuración avanzada:**

```json
{
    "editor.inlineSuggest.syntaxHighlightingEnabled": true,
    "editor.inlineSuggest.minShowDelay": 150,
    "editor.inlineSuggest.edits.allowCodeShifting": "auto",
    "editor.inlineSuggest.edits.renderSideBySide": "auto",
    "github.copilot.nextEditSuggestions.fixes": true
}
```

## Configuraciones de GitHub Copilot Chat

Las configuraciones de Copilot Chat permiten personalizar el comportamiento del chat integrado en VS Code, incluyendo la interfaz, el formato de respuestas, la ubicación de elementos y las opciones de accesibilidad. Estas configuraciones son esenciales para adaptar la experiencia de chat a las preferencias personales o a los requisitos del flujo de trabajo del equipo.

El parámetro `github.copilot.chat.localeOverride` es particularmente útil para desarrolladores que trabajan en equipos internacionales, ya que permite especificar el locale que Copilot Chat utilizará para generar respuestas. Esto es especialmente relevante cuando se desea que las explicaciones de código estén en un idioma específico, independientemente de la configuración regional del sistema operativo.

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `github.copilot.chat.localeOverride` | cadena | `"auto"` | Especifica el locale para las respuestas del chat. Valores comunes: `"en"`, `"es"`, `"fr"`, `"de"`, `"ja"`, `"zh"`. |
| `github.copilot.chat.useProjectTemplates` | booleano | `true` | Habilita el uso de proyectos de GitHub como proyectos iniciales cuando se usa el comando `/new`. |
| `github.copilot.chat.scopeSelection` | booleano | `false` | Solicita selección de ámbito de símbolo con `/explain` cuando no hay selección activa en el editor. |
| `github.copilot.chat.terminalChatLocation` | cadena | `"chatView"` | Controla donde se abren las consultas de chat del terminal. Valores: `"chatView"`, `"terminal"`, `"panel"`. |
| `chat.detectParticipant.enabled` | booleano | `true` | Habilita la detección automática de participantes de chat para dirigir consultas al agente correcto. |
| `chat.checkpoints.enabled` | booleano | `true` | Habilita los puntos de control en las sesiones de chat, permitiendo navegar entre estados de la conversación. |
| `chat.checkpoints.showFileChanges` | booleano | `false` | Muestra un resumen de cambios de archivos al final de cada solicitud en el chat. |
| `chat.editRequests` | cadena | `"inline"` | Habilita o deshabilita la edición de solicitudes de chat anteriores. Valores: `"inline"`, `"sidebar"`, `"off"`. |

### Configuraciones de Fuente y Apariencia del Chat

Estas configuraciones permiten personalizar la tipografía y apariencia del chat para mejorar la legibilidad según las preferencias del usuario.

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `chat.editor.fontFamily` | cadena | `"default"` | Familia de fuente utilizada en los bloques de código del chat. |
| `chat.editor.fontSize` | número | `14` | Tamaño de fuente en píxeles para el código en el chat. |
| `chat.editor.fontWeight` | cadena | `"default"` | Grosor de fuente para el código en el chat. Valores: `"normal"`, `"bold"`, números. |
| `chat.editor.lineHeight` | número | `0` | Altura de línea en píxeles para el código en el chat. El valor 0 usa el valor por defecto. |
| `chat.editor.wordWrap` | cadena | `"off"` | Activa el ajuste de línea en los bloques de código del chat. Valores: `"on"`, `"off"`, `"wordWrap"`. |
| `chat.fontFamily` | cadena | `"default"` | Familia de fuente para el texto Markdown en el chat. |
| `chat.fontSize` | número | `13` | Tamaño de fuente para el texto Markdown en el chat. |

### Configuraciones de Notificaciones del Chat

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `chat.notifyWindowOnConfirmation` | booleano | `true` | Muestra una notificación del sistema operativo cuando se requiere entrada del usuario en el chat. |
| `chat.notifyWindowOnResponseReceived` | booleano | `true` | Muestra una notificación cuando se recibe una respuesta del chat, útil para sesiones largas. |
| `chat.viewTitle.enabled` | booleano | `true` | Muestra el título de la sesión de chat en el encabezado (característica en Preview). |

**Ejemplo de configuración del chat:**

```json
{
    "github.copilot.chat.localeOverride": "es",
    "github.copilot.chat.useProjectTemplates": true,
    "github.copilot.chat.scopeSelection": true,
    "chat.checkpoints.enabled": true,
    "chat.editor.fontSize": 14,
    "chat.editor.wordWrap": "on",
    "chat.fontSize": 13,
    "chat.notifyWindowOnConfirmation": true,
    "chat.notifyWindowOnResponseReceived": false
}
```

## Configuraciones de Agentes

Las configuraciones de agentes permiten controlar el comportamiento de los agentes de IA de Copilot, incluyendo límites de solicitudes, auto-corrección y herramientas disponibles. Estas configuraciones son avanzadas y están diseñadas para usuarios que necesitan un control granular sobre las capacidades de los agentes.

El parámetro `chat.agent.maxRequests` establece un límite importante de seguridad que restringe el número máximo de solicitudes que Copilot puede realizar utilizando agentes en una sesión. Esto previene situaciones donde un agente podría generar un número excesivo de solicitudes debido a bucles infinitos o comportamientos inesperados.

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `chat.agent.enabled` | booleano | `true` | Habilita o deshabilita el sistema de agentes de Copilot (requiere VS Code 1.99 o superior). |
| `chat.agent.maxRequests` | número | `25` | Número máximo de solicitudes que Copilot puede realizar usando agentes por sesión. |
| `github.copilot.chat.agent.autoFix` | booleano | `true` | Habilita el diagnóstico y corrección automática de problemas en el código generado por los agentes. |
| `chat.mcp.access` | booleano | `true` | Gestiona el acceso de los agentes a servidores MCP (Model Context Protocol). |
| `chat.mcp.discovery.enabled` | booleano | `false` | Habilita el descubrimiento automático de configuraciones de servidores MCP. |

### Configuraciones de Auto-Aprobación de Herramientas

Estas configuraciones controlan qué comandos y herramientas pueden ejecutarse automáticamente sin confirmación del usuario, una característica de seguridad importante para prevenir acciones no deseadas.

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `chat.tools.terminal.autoReplyToPrompts` | booleano | `false` | Responde automáticamente a prompts del terminal sin intervención del usuario. |
| `chat.tools.terminal.enableAutoApprove` | booleano | `true` | Habilita la auto-aprobación de comandos del terminal especificados en la configuración. |
| `chat.tools.global.autoApprove` | booleano | `false` | Auto-aprueba todas las herramientas globalmente. **Advertencia:** Deshabilita las protecciones de seguridad. |
| `chat.tools.urls.autoApprove` | array | `[]` | Lista de URLs que pueden ser accedidas automáticamente sin confirmación. |
| `chat.tools.terminal.autoReplyToPrompts` | booleano | `false` | Permite que el agente responda automáticamente a solicitudes de entrada en el terminal. |

**Ejemplo de configuración de seguridad para agentes:**

```json
{
    "chat.agent.enabled": true,
    "chat.agent.maxRequests": 25,
    "github.copilot.chat.agent.autoFix": true,
    "chat.tools.terminal.enableAutoApprove": true,
    "chat.tools.global.autoApprove": false,
    "chat.tools.urls.autoApprove": [
        "https://api.github.com/",
        "https://docs.github.com/"
    ]
}
```

## Configuraciones de Inline Chat

El Inline Chat permite interactuar con Copilot directamente en el editor de código, sin necesidad de abrir el panel de chat lateral. Estas configuraciones controlan el comportamiento de esta modalidad de interacción.

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `inlineChat.finishOnType` | booleano | `false` | Finaliza la sesión de inline chat cuando el usuario escribe fuera de las regiones modificadas. |
| `inlineChat.holdToSpeech` | booleano | `true` | Mantener presionado el atajo para habilitar reconocimiento de voz en inline chat. |
| `inlineChat.lineEmptyHint` | booleano | `false` | Muestra una pista en líneas vacías para activar inline chat (Experimental). |
| `inlineChat.lineNaturalLanguageHint` | booleano | `true` | Activa inline chat automáticamente cuando se escribe lenguaje natural en líneas vacías (Experimental). |
| `inlineChat.accessibleDiffView` | cadena | `"auto"` | Renderiza el visor de diff accesible para los cambios realizados por inline chat. |

## Configuraciones de Instrucciones Personalizadas

Las instrucciones personalizadas permiten definir reglas específicas que Copilot debe seguir al generar código, evitando tener que repetirlas en cada prompt. Estas configuraciones controlan qué archivos de instrucciones se utilizan y cómo se aplican.

El parámetro `github.copilot.chat.codeGeneration.useInstructionFiles` determina si Copilot utiliza automáticamente las instrucciones definidas en el archivo `.github/copilot-instructions.md` del repositorio, una práctica fundamental para equipos que desean mantener consistencia en el estilo y convenciones del código generado.

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `github.copilot.chat.codeGeneration.useInstructionFiles` | booleano | `true` | Auto-agrega instrucciones desde `.github/copilot-instructions.md` a todas las generaciones de código. |
| `chat.instructionsFilesLocations` | objeto | `{ ".github/instructions": true }` | Ubicaciones de archivos de instrucciones personalizadas. Soporta múltiples paths. |
| `chat.promptFilesLocations` | objeto | `{ ".github/prompts": true }` | Ubicaciones de archivos de prompts reutilizables que pueden invocarse durante el chat. |
| `chat.promptFilesRecommendations` | array | `[]` | Archivos de prompts recomendados para mostrar al inicio de nuevas sesiones de chat. |
| `github.copilot.chat.commitMessageGeneration.instructions` | array | `[]` | Instrucciones personalizadas para la generación de mensajes de commit (Experimental). |
| `github.copilot.chat.pullRequestDescriptionGeneration.instructions` | array | `[]` | Instrucciones personalizadas para generación de descripciones de pull requests (Experimental). |

**Ejemplo de configuración de instrucciones personalizadas:**

```json
{
    "github.copilot.chat.codeGeneration.useInstructionFiles": true,
    "chat.instructionsFilesLocations": {
        ".github/instructions": true,
        ".copilot-instructions": true
    },
    "chat.promptFilesLocations": {
        ".github/prompts": true,
        ".prompts": true
    },
    "chat.promptFilesRecommendations": [
        "common-tasks.prompt.md",
        "code-review.prompt.md"
    ]
}
```

## Configuraciones de Archivos de Chat y Prompts

VS Code permite utilizar archivos de prompts reutilizables que se almacenan en el directorio `.github/prompts/` del proyecto. Estas configuraciones controlan el comportamiento de estos archivos.

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `chat.promptFiles` | booleano | `true` | Habilita el uso de archivos de prompts almacenados en `.github/prompts/` para invocarlos durante sesiones de chat. |
| `chat.useAgentsMdFile` | booleano | `true` | Utiliza archivos `AGENTS.md` como contexto adicional para las generaciones de código. |

**Estructura de archivos de prompts:**

Los archivos de prompts deben ubicarse en `.github/prompts/` y pueden invocarse durante el chat usando el nombre del archivo sin la extensión. Por ejemplo, un archivo `generate-tests.prompt.md` puede invocarse con `/generate-tests` en el chat.

```markdown
---
applyTo: "**/*.ts,**/*.tsx"
description: "Plantilla para generar pruebas unitarias"
---

Eres un experto en testing. Genera pruebas unitarias utilizando el framework de testing del proyecto.
Considera los siguientes patrones:
- Arrange-Act-Assert para cada test
- Mock de dependencias externas
- Cobertura de casos edge y errores
```

## Configuraciones de Accesibilidad

Las configuraciones de accesibilidad permiten adaptar la experiencia de Copilot para usuarios con diferentes necesidades, incluyendo señales de audio, reconocimiento de voz y navegación por teclado optimizada.

| Parámetro | Tipo | Valor por Defecto | Descripción |
|-----------|------|-------------------|-------------|
| `accessibility.signals.chatRequestSent` | objeto | `{ "sound": "auto", "announcement": "auto" }` | Señal cuando se envía una solicitud de chat. Valores: `"auto"`, `"on"`, `"off"`. |
| `accessibility.signals.chatResponseReceived` | objeto | `{ "sound": "auto" }` | Sonido cuando se recibe una respuesta del chat. |
| `accessibility.voice.keywordActivation` | cadena | `"off"` | Controla la activación por palabra clave «Hey Code» para iniciar sesiones de voz. |
| `accessibility.voice.autoSynthesize` | cadena | `"off"` | Lee automáticamente las respuestas en voz alta. |
| `accessibility.voice.speechTimeout` | número | `1200` | Tiempo de espera del reconocimiento de voz en milisegundos. |

## Configuración de Red y Proxy

Para entornos empresariales que requieren el uso de proxies, VS Code y GitHub Copilot ofrecen configuraciones específicas de red que deben configurarse adecuadamente para asegurar la conectividad con los servidores de GitHub.

| Parámetro | Tipo | Descripción |
|-----------|------|-------------|
| `http.proxy` | cadena | URL del proxy HTTP que VS Code debe utilizar para todas las conexiones de red. |
| `http.proxyStrictSSL` | booleano | Controla si el proxy requiere certificación SSL válida. |
| `github.copilot.advanced.useNetworkProxy` | booleano | Indica si Copilot debe utilizar la configuración de proxy del sistema. |
| `github.copilot.advanced.debug.overrideProxyUrl` | cadena | URL de proxy específica para las conexiones de Copilot (debug). |

**Ejemplo de configuración de proxy:**

```json
{
    "http.proxy": "http://proxy.empresa.com:8080",
```

## Configuración de Archivos Excluidos

VS Code permite configurar patrones de exclusión específicos para GitHub Copilot, evitando que procese archivos sensibles o que no deben ser considerados como contexto para las sugerencias de código.

| Parámetro | Tipo | Descripción |
|-----------|------|-------------|
| `github.copilot.filePatterns` | objeto | Patrones glob para archivos que Copilot debe ignorar. Valores: `true` (excluir), `false` (incluir). |

**Ejemplo de configuración de exclusiones:**

```json
{
    "github.copilot.filePatterns": {
        "**/secrets/**": true,
        "**/.env": true,
        "**/credentials.*": true,
        "**/config.local.*": true,
        "**/*.key": true,
        "**/*.pem": true,
        "**/appsettings.*.json": true,
        "**/Migrations/**": true
    }
}
```
