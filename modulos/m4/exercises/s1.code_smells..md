# Code smells en Gestión de usuarios

/speckit.analyze Analizar código PortalEmpleo para identificar code smells.

Stack: .NET 8, Clean Architecture
Referencia: .specify/memory/constitution.md (sección Coding Standards)

Categorías a analizar:
1. Long Method: métodos >30 líneas
2. Large Class: clases >500 líneas
3. Feature Envy: métodos que acceden más a datos de otras clases
4. Data Clumps: grupos de variables siempre usadas juntas
5. Primitive Obsession: uso excesivo de tipos primitivos

Herramientas a usar:
- Roslyn Analyzers (dotnet analyze)
- Complejidad ciclomática
- Métricas de acoplamiento

Artefacto a generar: docs/refactoring/code-smells.md
