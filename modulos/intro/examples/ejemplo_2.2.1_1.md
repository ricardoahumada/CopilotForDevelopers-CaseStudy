# Prompt Típico de Vibe Coding

**Contexto de uso:** Este ejemplo ilustra un prompt de Vibe Coding que demuestra las características típicas de este enfoque: descripciones en lenguaje natural, vagas y orientadas al resultado final sin detalles técnicos.

**Prompt original:**
```
"Quiero una página de login con email y password,
un botón de 'olvidé mi contraseña',
y que valide que el email tenga formato válido.
Hazlo con un diseño moderno y limpio."
```

**Características del prompt:**

| Aspecto | Descripción |
|---------|-------------|
| **Tipo** | Lenguaje natural conversacional |
| **Detalle técnico** | Ninguno especificado |
| **Framework** | No mencionado |
| **Criterios de éxito** | Subjetivos ("moderno y limpio") |
| **Validación** | Mínima ("email con formato válido") |

**Problemas identificados:**

| Problema | Descripción |
|----------|-------------|
| **Ambigüedad** | "Diseño moderno y limpio" es subjetivo |
| **Sin contexto técnico** | No especifica framework ni stack |
| **Validación vaga** | No define reglas exactas de validación |
| **Sin restricciones** | No indica qué NO hacer |
| **Scope creep** | El botón de "olvidé mi contraseña" añade complejidad |

**Lección aprendida:** Los prompts de Vibe Coding son rápidos pero producen resultados impredecibles. Para proyectos de producción, se necesitan especificaciones más detalladas.
